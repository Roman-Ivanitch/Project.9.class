Структура GET запроса
http://blynk-cloud.com/auth_token/update/pin?value=value

http:// - Если не хотим использовать SSL сертификат
https:// - Если хотм использовать SSL сертификат

blynk-cloud.com - имя сервера (blynk-cloud.com это родной сервер приложения) можно указать свой, но тогда 
нужно добавить порт. (Пример: Alex.dlinkddns.com:8080 или 192.168.0.1:8080) 

auth_token - Ваш токен из мобильного приложения приходит на почту

pin - пин (нога GPIO ) вашего устройства, но лучше использовать виртуальный пин

value - передоваемое значение
в принцепе 0 или 1, но ничего не мешает отправить 55 и обработать это как 55% яркости светодиода
нужно просто код написать в приложении. 

для Sonoff RF с моей прошивкой
http://blynk-cloud.com/auth_token/update/V0?value=1 	Включить V0
http://blynk-cloud.com/auth_token/update/V0?value=0 	Выключить V0

для ESP2866 (NodeMCU 1.0 (ESP-12E Module) с моей прошивкой
http://blynk-cloud.com/auth_token/update/V1?value=1 	Включить V1
http://blynk-cloud.com/auth_token/update/V1?value=0 	Выключить V1
http://blynk-cloud.com/auth_token/update/V2?value=1 	Включить V2
http://blynk-cloud.com/auth_token/update/V2?value=0 	Выключить V2
http://blynk-cloud.com/auth_token/update/V3?value=1	 Включить V3
http://blynk-cloud.com/auth_token/update/V3?value=0	 Выключить V3
http://blynk-cloud.com/auth_token/update/V4?value=1 	Включить V4
http://blynk-cloud.com/auth_token/update/V4?value=0 	Выключить V4
